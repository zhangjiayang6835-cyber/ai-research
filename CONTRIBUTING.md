# 🤝 贡献指南

## 如何参与

### 1. 认领任务
在任意 Open Issue 下评论 `/claim`，Bot 会自动分配给你。

### 2. 提交修复
有两种方式：

**方式 A: 创建 PR（推荐）**
1. Fork 本仓库
2. 创建分支，写修复代码
3. 提交 PR，标题包含关联 Issue 编号
4. Bot 会自动审查 + 评分 + 合并

**方式 B: 在 Issue 评论中贴代码**
直接在 Issue 下评论，用代码块（```python）粘贴修复代码，Bot 会自动评估。

### 3. 评分标准
| 维度 | 说明 |
|------|------|
| 代码安全 | 无 shell=True、eval() 等危险函数 |
| 修复完整性 | 覆盖所有攻击面 |
| 代码质量 | 有注释、有测试优先 |
| 提交速度 | 从 Issue 创建到修复提交的时间 |

### 4. 代币说明
> 💡 虚拟代币仅供学习排名使用，不可兑换为现金或加密货币。

### 5. 规则
- 每个 Issue 只能一人认领
- 认领后 7 天未提交将自动释放
- 禁止刷屏、重复提交
- 作弊检测通过的 PR 会被自动合并

## Bot 系统
本仓库由 15+ 个 GitHub Actions Bot 自动管理，包括：
- 自动审查 + 评分
- 自动合并干净 PR
- 每日/周/月报告
- 垃圾检测
- 过期任务清理
